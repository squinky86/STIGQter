\section{Background}

STIGQter is an open source tool used to perform RMF and STIG compliance mapping. As an open-source application, RA is performed by the STIGQter administrators on GitHub at \url{https://github.com/squinky86/STIGQter}.

\section{Purpose}

This documentation is meant to satisfy Risk Assessment (RA) requirements for RMF and detail what pieces of a PS policy may be covered by this application. RA requirements of roles (Section~\ref{sec:roles}) and their responsibilities (Section~\ref{sec:responsibilities}) support continuous assessment of the software.

\section{Scope}

The RA policy details the privacy responsibilities of administrators.

\section{Roles}
\label{sec:roles}

The following role(s) are defined as being applicable to STIGQter execution:
\begin{enumerate}
	\item \textbf{User} -- The user of the software is defined as the individual account given permissions to execute the software.
\end{enumerate}

The following role(s) are defined as being applicable to STIGQter GitHub administration:
\begin{enumerate}
	\item \textbf{Administrator} -- An administrator of the STIGQter GitHub project
\end{enumerate}

\section{Responsibilities}
\label{sec:responsibilities}

The Administrator role is responsible for any RA requirements listed here.

\section{Management Commitment}

STIGQter Risk Assessment policies are reviewed annually in an approved open source repository for maximum visibility and distribution.

\section{Coordination}

No coordination with other organizations is required to operate this software.

\section{RMF Control Compliance}

All personnel security requirements take place full and open in the Github repository. The nature of open source software is to provide maximum visibility into the RA process.

\subsection{RA-5 -- Vulnerability Scanning}

\paragraph{Applicable CCIs:} CCI-1057, CCI-1058, CCI-1059, CCI-1060, CCI-1061, CCI-2376

\paragraph{Non-Applicable CCIs:} CCI-1054, CCI-1055, CCI-1056, CCI-1641, CCI-1643

The software does not perform vulnerability scanning on the platforms on which it is installed. The assessment is conducted below the information system level. The application is not a hosted application.

Vulnerability analysis procedures and reporting occurs publicly. All scan data is available via the Github portal and triaged by the STIGQter administrators at least weekly. There are no restrictions for accessing scan data. Please see the SA policy for specifics regarding the continuous vulnerability scanning of the application.
