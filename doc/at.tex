\section{Background}

STIGQter is an open source tool used to perform RMF and STIG compliance mapping.

\section{Purpose}

This documentation is meant to satisfy Awareness and Training (AT) requirements for RMF and detail what pieces of an AT policy may be covered by this application.

\section{Scope}

The AT policy details the Awareness and Training requirements for personnel involved with STIGQter.

\section{Roles}
\label{sec:roles}

The following role(s) are defined as being applicable to STIGQter execution:
\begin{enumerate}
	\item \textbf{User} -- The user of the software is defined as the individual account given permissions to execute the software.
\end{enumerate}

The following role(s) are defined as being applicable to STIGQter GitHub administration:
\begin{enumerate}
	\item \textbf{Administrator} -- An administrator of the STIGQter GitHub project
\end{enumerate}

\section{Responsibilities}
\label{sec:responsibilities}

Responsibilities are described in the GPL version 3.

\section{Management Commitment}

STIGQter development is committed to occur in an approved open source repository for maximum visibility and distribution.

\section{Coordination}

No coordination with other organizations is required to operate this software.

\section{RMF Control Compliance}

\subsection{AT-3 -- Role-Based Security Training}
\label{sec:at-3}

\subsubsection{AT-3(2) -- Physical Security Controls}

\paragraph{Applicable CCIs:} None

\paragraph{Non-Applicable CCIs:} CCI-1566, CCI-1567, CCI-1568, CCI-2051

The application is open-source and publicly available, and therefore does not enforce organization-defined security training protocols for physical security of the application hardware. The application assumes that the user possesses the knowledge and ability to ensure the physical security of the application hardware.

\subsubsection{AT-3(3) -- Practical Exercises}

\paragraph{Applicable CCIs:} CCI-2052

The software is publicly released; no training is required as a user or administrator. Annually, each administrator must review GPL compliance for the application.
