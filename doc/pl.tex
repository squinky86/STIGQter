\section{Background}

STIGQter is an open source tool used to perform RMF and STIG compliance mapping. As an open-source application, PL is performed by the STIGQter administrators on GitHub at \url{https://github.com/squinky86/STIGQter}.

\section{Purpose}

This documentation is meant to satisfy Planning (PL) requirements for RMF and detail what pieces of a PL policy may be covered by this application. PL requirements of roles (Section~\ref{sec:roles}) and their responsibilities (Section~\ref{sec:responsibilities}) maintain and define the security assessments for the system.

\section{Scope}

The PL policy details the planning responsibilities of administrators.

\section{Roles}
\label{sec:roles}

The following role(s) are defined as being applicable to STIGQter execution:
\begin{enumerate}
	\item \textbf{User} -- The user of the software is defined as the individual account given permissions to execute the software.
\end{enumerate}

The following role(s) are defined as being applicable to STIGQter GitHub administration:
\begin{enumerate}
	\item \textbf{Administrator} -- An administrator of the STIGQter GitHub project
\end{enumerate}

\section{Responsibilities}
\label{sec:responsibilities}

The Administrator role is responsible for planning.

\section{Management Commitment}

STIGQter planning is reviewed annually in an approved open source repository for maximum visibility and distribution.

\section{Coordination}

No coordination with other organizations is required to operate this software.

\section{RMF Control Compliance}

All planning takes place full and open in the Github repository. The nature of open source software is to provide maximum visibility into the planning process.

\subsection{PL-2 -- System Security Plan}

\paragraph{Applicable CCIs:} CCI-571, CCI-572, CCI-573, CCI-574, CCI-3049, CCI-3050, CCI-3053, CCI-3056, CCI-3057, CCI-3059, CCI-3060, CCI-3061, CCI-3062, CCI-3063, CCI-3064

Categorization and tailoring are conducted using due diligence of the STIGQter administrators. No overlays apply, and the application is not the protection mechanism for any particular information type. Only the security controls listed in all policies linked to in the User's Guide are applicable to the application. In eMASS, the security plan is approved by the AO. While the security plan from eMASS is not available publicly, it reflects the controls documented in this publicly available documentation and is available to all.

Changes to the security plan posture occur with the updated documentation in each STIGQter release. The public plan is updated with each release. All people are allowed to fork STIGQter and modify their own security plans; however, the official STIGQter release is modifiable only by the STIGQter administrators.

\paragraph{Non-Applicable CCIs:} CCI-3051, CCI-3052, CCI-3054, CCI-3055

An assessment below the system level is not an authorization. The operational context of systems that incorporate the STIGQter assessment are outside of the scope of planning.
